# MaxLeap FAQ


## 平台咨询

### MaxLeap 部署在哪个云平台上
MaxLeap 部署在国内多个云计算平台上，并采用在双线机房内同时使用虚拟机和实体机的混合部署策略，来保证应用的访问体验和可靠性
### MaxLeap 国内外是否都有节点
MaxLeap 国内和海外都有节点，但两个节点数据不互通

国内用户尽量使用国内节点
### 企业版和非企业版有何区别
企业版可根据客户特殊需求量身定制，可进行私有部署，并有专属销售及客服为您服务

非企业版主要是公有云模式，用户注册并使用 MaxLeap 已经提供的服务
### MaxLeap 数据是开放的么
是开放的，您可自由导入导出
### MaxLeap 支持一个团队不同角色成员管理不同模块么
必须支持的，注册用户（管理员）可以在【账号中心 -> 团队管理】，创建不同角色成员，并分配给他们管理不同应用以及对应的模块
### 提供哪些平台的 SDK
目前 MaxLeap 提供如下平台 SDK

* iOS
* Android
* Javascript
* Java
* Python
* Node.js
* PHP

更多平台 SDK 正在开发中，敬请期待

### iOS 和 Android 是否可以使用同一个 App

MaxLeap 平台上应用是不区分操作系统的，支持同一个应用开发多个平台的版本，共享后端数据。

## 费用相关
### 非企业版计费是基于账号还是应用
MaxLeap 计费都基于用户账号，根据用户购买的服务版本收费，详细信息请参考 [价格](https://maxleap.cn/s/web/zh_cn/devcenter-pricing.html) 页面。

### 非企业版如果没有缴费会怎么样
账单逾期一个月未缴，账户服务将被停止；但仍会在 MaxLeap 云端保留一个月。如需要恢复服务和访问应用数据，请登录控制台，支付欠款。

我们为账单支付提供一段缓冲期，请在收到账单的十天内完成缴费即可。在此期间，我们会通过邮件或者电话等方式与您联系，因此强烈建议完整填写 开发者信息，以免错过我们的提醒和最佳付款时机。

### 账户余额有什么作用
您可以通过支付宝充值账户余额，然后使用账户余额购买短信或者购买 MaxLeap 服务版本

根据您选择的服务版本，购买对应版本并付费，支持账户余额和第三方支付渠道同时购买，目前支持的第三方支付渠道有：

* 支付宝

### 如何申请开具发票
支持，申请开发票前，请先按系统要求，完善所有必填的用户信息。

## 数据源
### 数据源和数据存储服务有什么区别
##### 1、数据源服务是是什么鬼
    
   数据源服务是 MaxLeap 为用户提供的原生第三方主流数据库服务，比如 Mysql,Mongo等；对用户来说，相当于在 MaxLeap 平台上申请一个 Mysql（举个例子）数据库，但不需要做运维方面的工作，和在本地使用 Mysql 数据库操作完全一致；唯一不同就是一个是放在 MaxLeap 平台上，一个是本地环境自行搭建并自行运维

**优势**

 * 0 学习成本，原来在本地怎么操作还是怎么操作
 * 无需运维
 * 数据无缝支持迁入迁出

**不足**

 * 所有数据库访问都得自行实现
 * 移动端无法直接访问，需要搭建服务端程序支持

#####2、数据存储服务是什么鬼
   
   数据存储服务是 MaxLeap 在已有主流数据库之上抽象出来的一个数据库，定义数据结构、关系、类型及存储格式，但基本和目前主流数据库定义相同，并提供 SDK 直接访问和在线控制台管理，提供更符合移动端数据存储业务需求的数据类型，比如可以存储地理位置、视频文件等，可以认为是一个偏向 SaaS 的数据存储服务。对用来说，这是一个新的数据库，需要适当学习才能使用

**优势**

 * 通用数据库访问无需自行实现，直接调用 SDK 访问，降低时间和人力成本
 * 无需运维
 * 数据支持迁入迁出

**不足**

 * 适当学习成本
 * 数据迁移至其他平台数据库需做适配

#####3、区别

* 数据存储服务是 MaxLeap 在已有主流数据库之上抽象出来的一个新数据库，并提供操作规范及帮助用户运维。


* 数据源服务是 MaxLeap 给用户提供一个原生数据库，仅帮助用户运维。

## 云容器
### 云容器和云代码有什么区别
##### 1、云容器是什么鬼
云容器服务是 MaxLeap 帮助用户部署及运维其后端应用程序的代码托管服务，用户只需要提供提供服务端的业务逻辑，包括静态网站或者动态应用程序。

对用户来说，仅仅是部署环境不同。

**优势**

 * 0 学习成本，应用程序代码无需任何修改，仅换个环境部署
 * 无需运维，无需考虑负载均衡、灰度发布等
 * 应用程序无缝支持迁入迁出

**不足**

 * 所有业务逻辑都得自行实现，包括通用数据库访问逻辑
 * 移动端网络访问逻辑自行实现

##### 2、云代码是什么鬼
云代码是 MaxLeap 抽象出来的服务端程序框架并运行在 MaxLeap 平台上，提供通用业务逻辑代码，还提供包括移动端在内的 SDK 直接访问；与 MaxLeap 数据存储服务深度集成，提供通用数据库访问逻辑，用户仅需自行实现其具体业务逻辑代码。

对用户来说，这是一个新的编程模式，需要适当学习

**优势**

 * 通用业务逻辑无需自行实现，直接调用 SDK 访问，降低时间和人力成本
 * 无需运维，无需考虑负载均衡、灰度发布等
 * 丰富功能及 API 直接调用

**不足**

 * 新的编程模式，适当学习成本
 * 应用程序迁至其他环境需适配

##### 3、区别

* 云容器是 MaxLeap **提供的一个部署环境** 给用户部署其服务器应用程序
* 云代码是 MaxLeap **提供的一种新的编程模式及环境**，用户在 MaxLeap 的服务框架内实现服务端逻辑，最后并部署至 MaxLeap 平台

## 移动支付

### 使用 MaxLeap 移动支付有哪些条件

* 申请好第三方支付渠道账户
* 具备基础开发能力

### 是否支持个人开发者？
支持。只要你能在第三方支付渠道（例如支付宝，微信支付等）开通账号。

### MaxLeap 移动支付费用怎么算
MaxLeap 不收取任何费用，各支付渠道收费情况根据对应渠道的要求

### WebHook 是什么东西，是否必须配置
为了便于您的系统处理交易成功信息， MaxLeap 提供 WebHook 功能，可以按照您的要求把特定的事件结果推送到指定的地址以便于您做后续处理

比如您的业务逻辑是，用户支付成功后给需给其发送短信或者邮件，那么您此时需要配置 WebHook；发送短信或者邮件的业务逻辑，您可以自行搭服务器实现或者使用 MaxLeap 云函数

如果您自行搭建服务器还发送短信或者邮件，那您就需要配置自定义 URL 形式 WebHook，来接收 MaxLeap 发送的支付成功事件，成功接收后，发送短信或者邮件

如果您是使用 MaxLeap 云代码来实现发送短信或邮件，那么您只需要选择对应的 MaxLeap 云函数作为 WebHook，来接收支付成功事件，成功接收后，发送短信或者邮件

假如您的业务在支付成功后无任何其他业务，则不需要配置 WebHook

需要强调的是，**WebHook 不是必须配置的**
